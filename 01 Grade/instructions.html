Assignment 1: Grade Converter [0%]

This assignment has zero weighting (it doesn't contribute to your unit mark).
You should however still submit it as it will be good practice to do so (both for you and the marking team).

Read all the instructions before attempting anything (there may be important details near the end !)
This is a relatively simple assignment, just to ease you into programming in Java.
The "hidden agenda" of this assignment is to thrash out any problems with the installation of java
(both in the lab and on your own computer)

Overview

The aim of this assignment is to create a program to convert a mark (between 0 and 100)
into a grade ("Fail", "Pass", "Merit", "Distinction").
If you don't know the grade boundaries for your degree, maybe now is the time to look them up.
You will be given a template/skeleton project to help get you started.
You must add to the GradeConverter class and complete the following methods:

  • convertStringToMark: Reads a mark from a String and returns the mark as an int (0 to 100)
  • convertCharToInt: Converts a single character to an int - this will be used by convertStringToMark
  • convertMarkToGrade: Converts a numerical mark (e.g. 68) into a textual grade (e.g. "Merit")

    You may find it useful to take a look at the documentation for the Java String class

    Conversion

    Do NOT use existing libraries/methods to convert Strings to ints (that would make it too easy).
    Instead, write your own function (convertCharToInt) that converts a single character into an
    integer between 0 and 9 (or -1 if the character is not a numerical digit).
    Don't use a bunch of IF statements for this - instead use a calculation based on ascii values.

    Running your Code

    A Makefile is provided to help you: type make to compile your code and make run to run it.
    The loop in the main method will repeatedly prompt the user for a mark
    and will then use your code to convert it into a grade, before printing it out.

    Testing your Code

    There is a separate test class (GradeConverterTester) that contains a range of different test cases.
    Type make test to run the test cases against the current version of your code.
    Extend and improve your code bit-by-bit in order to pass one test at a time
    (don't try to pass them all in one go at the start !)
    Your eventual aim is to pass as many tests as possible.

    Robustness

    You should make your code robust (so that it does something sensible if an invalid input is given).
    As much as possible, try to deal elegantly with unexpected input.
    For example, if the user enters a % symbol at the end of their input, your code should still convert the mark.
    There are many other weird inputs you might get from the user - try to anticipate these.

    Submission

    This assignment is not assessed, however it will be good practice to submit it onto blackboard
    (go to "Assessment, submission and feedback" and then "Other Formative and Draft Submissions").

    It is essential that you ensure your code compiles and runs before you submit it
    (otherwise we will not be able to run it to mark it !)
    Just Submit your GradeConverter class (nothing else should be needed)
    Scripts will be used to automatically test your code to make sure it operates correctly.
    Don't change the name of the class or the signatures of any of the methods that already exist
    (or we won't be able to test your code !).
